설계 원칙

7 RULES
의미색은 겹치지 않는다

green = 선택·통과·주 액션, amber = 연결·주의, red = 제외·부정, purple = 파생·메타. 새 상태가 필요하면 색을 재활용하지 말고 팔레트 추가를 먼저 논의한다.

그림자를 쓰지 않는다

높이는 표면색 + 1px 보더로만. box-shadow는 전부 inset 마커(셀 링·행 마커·내비 마커·밑줄)이며 높이를 만들지 않는다.

mono는 신호다

기계가 생성했거나 열로 정렬돼야 하는 것에만 붙인다. 우측 정렬 숫자는 예외 없이 mono.

표는 빽빽하다

기본 셀 패딩 5–5.5px는 화면당 행 수를 최대화하는 의도된 값. 밀도는 .tbl-compact/.tbl-roomy로만 조절한다.

곡률이 성격을 말한다

시트 격자 0 → 컨트롤 2px → 표면 3px → 검색·자연어 입력 20px. 데이터에서 멀어질수록 둥글어진다.

sparse ≠ zero

빈 값은 0이 아니라 –. 값 없음과 0을 시각적으로 구분하는 것이 이 시스템의 핵심 계약이다.

다크는 기계의 것

다크 표면(ink 배경)은 코드·스펙·로그·시스템 토스트 등 기계가 쓴 것에만 쓴다. 강조를 위해 일반 UI를 다크로 만들면 이 신호가 희석된다.

AI 참조 · 설치

llms.txt

외부 AI(코드 생성 도구·에이전트)가 이 시스템으로 UI를 만들 때는 llms.txt 하나만 컨텍스트에 넣으면 된다 — 토큰·클래스 API·규칙만 담은 도메인 중립 스펙이라, 특정 업무 화면의 문구나 데이터에 과적합되지 않는다. 문서 페이지들은 사람을 위한 해설이다.

설치 — npm i -D tailwindcss @tailwindcss/cli 후 앱 진입 CSS에서 @import "green-sheet-design-system/theme.css"; 하나만 import 하면 된다.
