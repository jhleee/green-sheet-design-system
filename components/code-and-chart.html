이 시스템에서 다크 표면은 딱 한 가지를 뜻한다 — 기계가 쓴 것. 설정 스펙, 생성된 쿼리, 로그가 전부다. 표면의 배경은 ink(#1B2730) 그 자체다. 종이에 잉크를 얹는 게 아니라, 잉크 위에 쓴다 — 라이트 UI를 뒤집은 것이 아니라 같은 잉크의 다른 면이다.

스펙 블록 시스템 제안 → 사람 승인 
# import_spec: sales_2026_q1.yaml
source: uploads/sales_q1.xlsx
grain: month                     # 최소 단위 사실
header: {rows: [1], type: single}
columns:
  region → dim                  # 차원
  qty → measure(SUM)            # 측정값
exclude_rows:
  pattern: "(소계|합계)"
  use_for: validation           # 버리되 checksum으로 활용
on_empty_cell: skip             # sparse ≠ zero

3색만 쓴다 — 키워드 green-bright, 문자열 amber-code, 주석 code-dim. 라이트 UI의 의미색이 다크에서도 같은 뜻을 유지한다: green은 여전히 구조, amber는 여전히 리터럴 값이다.

생성된 쿼리 + 규칙 칩 TEXT → SQL
GENERATED SQL
SELECT period, region, SUM(qty) AS total_qty
FROM fact_sales
WHERE dataset = 'sales_2026_q1'
  AND status = 'active'
GROUP BY period, region
ORDER BY period;
✓ 소계 미적재 → 이중집계 없음 ✓ qty는 SUM 허용 지표 ✓ 제외 행 필터 적용

규칙 칩은 쿼리가 왜 그렇게 생성됐는지를 밝힌다. 결과만 보여주는 대신 근거를 남겨야 사람이 승인할 수 있다. green 계열 반투명이라 다크 위에서도 "검증됨"이라는 뜻이 그대로 유지된다.

스키마 덤프가 아니라 의미 계층이다. 동의어·집계 규칙·파생 수식을 남겨야 사람도 기계도 잘못 집계하지 않는다. 카드의 배지 색이 곧 그 지표의 성격이다 — green은 그냥 더하면 되는 measure, purple은 파생, amber는 조심해서 다뤄야 하는 것.

⚠ 2026은 진행중 기간(YTD)입니다. period_type 규칙에 따라 완결 연도와의 직접 비교에서 자동으로 플래그 처리되었습니다.

차트에는 전용 토큰이 없다. 시리즈는 green → amber → purple 순서로 의미 팔레트를 그대로 쓴다 — 차트가 나머지 UI와 같은 언어를 말하게 하기 위함이다. 그리드라인은 표의 셀 격자와 같은 line-soft, 축 눈금은 보조 텍스트와 같은 ink-soft(흰 배경 대비 5.29:1, WCAG AA 통과). 부분 기간 같은 신뢰도 플래그는 색이 아니라 투명도 + 점선 테두리 + 라벨로 표시한다 — 색만으로 경고하면 색각 이상 사용자에게는 경고가 전달되지 않는다. SVG 안의 색은 유틸리티가 닿지 않으므로 리터럴 hex를 쓰되, 값은 전부 토큰과 동일하다.

규칙 — 다크 표면은 기계가 쓴 것에만 쓴다. 시각적 강조가 필요하다는 이유로 일반 UI를 다크로 만들지 않는다. 그러면 "기계가 쓴 것"이라는 신호가 희석되고, 다크 표면은 그냥 또 하나의 장식이 된다.
