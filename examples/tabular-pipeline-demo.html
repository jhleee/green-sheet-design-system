Ledger → Long Format → Semantic SQL

장표는 무한히 다양해도,
목표 모델은 하나로 수렴한다

간편장부, 손익계산서, 연결재무제표 — 서로 다른 세 장표가 파싱 스펙을 거쳐 동일한 fact 테이블로 정규화되고, 시맨틱 카탈로그를 통해 LLM 쿼리로 이어지는 과정을 실제 데이터로 시연합니다.

STEP 1파싱 스펙LLM 제안 → 사람 승인
STEP 2정규화 적재long 포맷 fact
STEP 3정합성 검증장부 내장 checksum
STEP 4시맨틱 쿼리text → SQL → 차트

원본 장표 ↔ Fact 테이블 매핑 INTERACTIVE

왼쪽 원본의 값 셀에 마우스를 올리면 오른쪽 fact 테이블에서 대응 행이 강조됩니다(반대 방향도 동일). 붉은 취소선 행은 파생 가능한 소계라 적재에서 제외되고 검증에만 쓰입니다.
