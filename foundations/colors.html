다크 모드 OPT-IN · data-theme="dark"

병합 이력 36 → 24 → 25

원본 데모에는 토큰 36개(고유값 30)가 있었다. 그중 12개를 제거했다 — 대부분은 눈으로 구분되지 않는 중복이었고, 일부는 순수한 별칭이었다. 두 건(chart-axis, amber-mark)은 병합이 곧 접근성 수정이기도 했다 — 원본 값이 WCAG 대비에 미달했다. v0.2에서 code-bg는 다크 모드를 위해 다시 분리됐고 (라이트 값은 ink와 동일), code-dim(3.65→5.03:1)과 red(흰 텍스트 4.38→5.32:1)는 접근성 수정으로 값이 바뀌었다.

제거됨 대체 이유
panel-raised #FBFCFA → panel #FFFFFF ΔE≈2, 화면에서 안 보임. 패널 헤더는 border-bottom이 이미 구분한다
grid-head-alt #F3F6EF → paper #F4F6F1 ΔE≈1, 사실상 같은 색
line-row #EEF1EC → line-soft #E4E9E1 행 구분선과 셀 격자는 같은 무게여야 한다
chart-grid #E7ECE5 → line-soft #E4E9E1 ΔE≈1.5. 차트 그리드가 셀 격자와 다를 이유가 없다
chart-axis #8A98A2 → ink-soft #5C6E7A 대비 2.96:1 → 5.29:1. 원본은 y축 #8A98A2, x축 #5C6E7A로 축 색이 불일치했다 접근성 수정
green-chip #9BDDBB → green-bright #7FD1A7 둘 다 "다크 위 밝은 초록"
amber-mark #B98A1E → amber-ink #8A6414 대비 3.13:1 → 5.37:1. 원본 값은 AA-large는 통과했으나 실제 렌더 크기(9px)에서 미달이었다 접근성 수정
amber-row #FDEFE0 → amber-soft #FBF1DA ΔE≈4
code-bg #182430 → ink #1B2730 ΔE≈1.5. 별개의 색이 아니었다
chart-1 / chart-2 / chart-3 → green / amber / purple 순수 중복 별칭
